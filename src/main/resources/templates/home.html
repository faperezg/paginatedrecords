Prueba desarrolladores Java Azurian
Requerimientos

Implementar los siguientes componentes utilizando Maven para la gestión de dependencias:

Backend

Implementar la consulta de una lista de registros desde una base de datos y disponibilizar dicha operación como servicio REST y/o SOAP. Herramientas a utilizar:

  • Java/Spring
  • JPA
  • Base de datos, para lo cual debe utilizar alguna de las siguientes alternativas:
    • MySQL (debe adjuntar scripts para creación de tablas e inserción de data inicial)
    • Base de datos en memoria. (Ej: HSQL, H2, Derby)
Frontend

Implemente una aplicación web con la(s) tecnología(s) que estime conveniente (Java/JSP, Java/JSF, Angular, etc) y que haga uso de los servicios implementados en backend (SOAP o REST) para desplegar el listado de registros en una tabla paginada.

Descripción de la solución

Se implementaron dos componentes, los cuales se describen a continuación:

  • Componente MVC que muestra esta página. Se accede en la raíz del contexto donde se haya instalado la aplicación
  • Componente REST que ofrece las operaciones de creación/lectura/actualización/eliminación sobre un repositorio en memoria con data falsa de Personas obtenida de internet. Se accede en la ruta /api/people a partir del contexto donde se haya instalado la aplicación

Para ello se tomaron las siguientes decisiones

  • Desarrollar un servicio web REST, dejando de lado el componente SOAP. REST es una tecnología más simple, lo que permite desarrollar más elementos para esta prueba, y así mostrar más capacidades
  • El servicio web soportará la propuesta HATEOAS para definir la estructura de la información recibida del servidor
  • Utilizar Spring Boot como punto de partida en lugar de Spring puro. Spring Boot es un framework ligero que funciona sobre Spring (por lo que no se rompe el requerimiento inicial de usar Spring) y ofrece una serie de elementos al programador que reducen la cantidad de código necesario, así como el tiempo requerido para completar la solución, algo necesario ante la premisa de entregar en menos de 3 días.
  • Utilizar Hibernate como implementación JPA. Hibernate ofrece un ORM robusto, tiene una amplia comunidad de desarrolladores, sigue en constante evolución, es fácil de configurar y de utilizar
  • Utilizar H2 como base de datos. No requiere configuración especial, la prueba es bastante simple, y tenía un mínimo de experiencia usándola. MySQL requiere de un servidor aparte, archivos adicionales, y no ofrece a objeto de la prueba ninguna ventaja.
  • Utilizar BootstrapCSS 4 como herramienta para la generación del frontend. Es muy simple de usar, tengo suficiente experiencia, ofrece herramientas muy versátiles para con poco código armar un frontend aceptable
  • Utilizar KnockoutJS como framework MVVM. Mis fortalezas en el desarrollo de software están del lado del Backend, y no tengo experiencia alguna con frameworks o librerías más modernas (AngularJS, ReactJS, VueJS).

El paquete tld.faperezg.paginatedrecords consta de lo siguiente:

  • Entidad Person, con los atributos id, Nombre y apellido de la persona, edad, número de pasaporte y dirección, con su correspondiente repositorio
  • Servicio PersonService para gestionar las operaciones de negocio sobre la entidad
  • Controlador MVC HomeResource para desplegar esta página
  • Controlador REST PersonResource que ofrece los puntos de acceso del servicio web
  • Gestor de excepciones para convertir a JSON los mensajes de error
  • Clases utilitarias para convertir fácilmente de los objetos entidad a objetos compatibles con HATEOAS
  • Pruebas unitarias y funcionales
